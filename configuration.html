General Settings

The General section is where you configure how Uplink connects to OpenClaw. The key setting is the Gateway URL â€” this points to your running OpenClaw gateway.

example 
Gateway URL: http://localhost:5001/v1
Model:       openclaw/default
Temperature: 0.7

The gateway URL is auto-detected if OpenClaw is running on the same machine. If you're running OpenClaw on a different host (or via Tailscale), update this to point to the correct address.

Security Settings

Uplink supports optional end-to-end encryption with AES-256. When enabled, all conversations are encrypted at rest on the server.

  • Enable Encryption â€” Toggle E2E encryption on/off
  • Set Password â€” Create or change your encryption password
  • Session Timeout â€” Auto-lock after inactivity (5 min, 15 min, 1 hour, never)
âš  Password Recovery If you enable encryption and forget your password, your locally stored conversations cannot be recovered. Uplink uses zero-knowledge encryption â€” the server never sees your plaintext password.

Environment Variables

Some settings can also be set via environment variables before starting the server. These override the defaults but can still be changed in the Settings panel.

bash 
# Set the port (default: 3456)
PORT=3456 node server.js

# Set the gateway URL
GATEWAY_URL=http://localhost:5001/v1 node server.js

# Enable debug logging
DEBUG=true node server.js

Multi-Device Sync

Uplink syncs conversations across devices via WebSocket. Any device connected to the same Uplink server sees the same conversations in real-time. There's no separate sync configuration â€” it works automatically.
