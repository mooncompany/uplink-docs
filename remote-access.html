Remote Access

Access your Uplink instance from any device ‚Äî phone, tablet, or another computer.

How It Works

Uplink is a web server. By default it listens on localhost:3456, which means only the machine running it can access the UI. To use Uplink from other devices, you need to either access it over your local network (LAN) or set up a secure tunnel for remote access.

LAN Access

The simplest option. If your phone/tablet/laptop is on the same Wi-Fi network as the machine running Uplink, you can access it directly via the host's LAN IP.

Find Your LAN IP

Windows PowerShell 
ipconfig | findstr "IPv4"
# Example output: IPv4 Address. . . : 192.168.1.100
macOS / Linux 
hostname -I  # Linux
ipconfig getifaddr en0  # macOS
# Example output: 192.168.1.100

Connect from Another Device

Open a browser on your phone or other device and navigate to:

url 
http://192.168.1.100:3456

Replace the IP with your actual LAN address. That's it ‚Äî you'll see the full Uplink interface. Conversations sync in real-time via WebSocket.

üì± PWA Tip On mobile, use your browser's "Add to Home Screen" option to install Uplink as a Progressive Web App. It'll look and feel like a native app ‚Äî full screen, no browser chrome.

Tailscale (Recommended for Remote)

For accessing Uplink outside your home network, Tailscale is the recommended approach. It creates a secure mesh VPN between your devices with zero configuration ‚Äî no port forwarding, no dynamic DNS, no firewall rules.

Setup

  1. Install Tailscale on the machine running Uplink
  2. Install Tailscale on your phone/laptop/remote device
  3. Sign in with the same account on both devices
  4. Access Uplink using the Tailscale IP (usually 100.x.y.z:3456)
bash 
# Check your Tailscale IP
tailscale ip -4
# Example: 100.64.0.1

# Access Uplink from any Tailscale-connected device:
# http://100.64.0.1:3456

Network Binding

By default, Uplink binds to 127.0.0.1 (localhost only) when authentication is disabled. This is a security measure ‚Äî but it also means Tailscale and LAN devices can't connect. You need to tell Uplink to listen on all interfaces:

bash 
# Option 1: Environment variable
UPLINK_HOST=0.0.0.0 uplink-chat start

# Option 2: PowerShell (Windows)
$env:UPLINK_HOST="0.0.0.0"; uplink-chat start

# Option 3: Add to your .env file
UPLINK_HOST=0.0.0.0
üí° Tip Enabling authentication (UPLINK_AUTH_ENABLED=true) automatically binds to 0.0.0.0, so you won't need to set UPLINK_HOST separately.

Tailscale HTTPS

When Uplink detects a Tailscale connection, it automatically provisions a TLS certificate and starts an HTTPS server on port 3457. This enables microphone and camera access on mobile (browsers require HTTPS for media APIs).

url 
# Tailscale HTTPS (enables mic/camera on mobile)
https://your-hostname.tailnet-name.ts.net:3457

# Plain HTTP (no mic/camera on mobile)
http://100.x.x.x:3456

Gateway URL Must Stay Localhost

A common mistake: setting GATEWAY_URL to your Tailscale IP instead of localhost. The OpenClaw gateway only listens on 127.0.0.1 ‚Äî Uplink proxies requests to it locally, regardless of how remote clients connect to Uplink.

.env 
# ‚úÖ Correct ‚Äî gateway is always local
GATEWAY_URL=http://127.0.0.1:23248

# ‚ùå Wrong ‚Äî gateway doesn't listen on Tailscale IP
GATEWAY_URL=http://100.85.40.53:23248

If GATEWAY_URL points to a non-localhost address, you'll see the Uplink UI load normally but messages will fail with "No response received."

Firewall Notes

If devices on your LAN can't connect, your firewall may be blocking port 3456. Here's how to allow it:

Windows PowerShell (Admin) 
New-NetFirewallRule -DisplayName "Uplink" -Direction Inbound -Port 3456 -Protocol TCP -Action Allow
Linux (ufw) 
sudo ufw allow 3456/tcp
‚ö† Security Warning Never expose Uplink directly to the public internet via port forwarding. It's designed for trusted networks. If you need internet access from anywhere, use Tailscale or another VPN. Uplink does not have built-in authentication for the web UI (encryption protects message content, not access).

Multi-Device Sync

All devices connected to the same Uplink server share conversations in real-time. When you send a message from your phone, it appears instantly on your desktop ‚Äî and vice versa. This is handled automatically via WebSocket; no additional configuration needed.
